# 引入：班里排队，怎么又快又整齐？

## 一个生活场景

新学期开学，班主任说：**「同学们按身高从矮到高排成一列！」**

30个同学挤在走廊里，怎么排？

### 方法1：一个一个比（冒泡思路）
> 相邻两个人比一比，矮的站前面，高的站后面。从头到尾走一遍，最高的就"浮"到最后了。再来一遍、再来一遍……直到所有人都站对。

### 方法2：挑最矮的站前面（选择思路）
> 扫一眼所有人，找出最矮的，让他站第一个。再从剩下的人里找最矮的，站第二个……

### 方法3：后来的人插到正确位置（插入思路）
> 已经站好的队伍是有序的，新来的人直接从前往后找，找到比自己高的第一个人，插到他前面。

**这三种方法都能把队伍排好，但快慢不一样！**

# 基础概念

## 排序是什么？

### 专业版本

> **排序（Sorting）**是将一组**无序**的数据元素，通过比较和交换，调整为**按关键字有序**的序列的过程。

### ~~人话版本（bushi）~~

**就是把一堆乱七八糟的东西，按某个规则（从小到大/从大到小/按字母顺序……）排整齐。**

## 排序算法的两个关键指标

| 指标 | 含义 | 人话 |
|------|------|------|
| **时间复杂度** | 算法运行时间随数据量增长的速度 | 数据翻倍，时间翻几倍？ |
| **稳定性** | 相同关键字的元素，排序后相对位置是否不变 | 两个并列第一，谁排前面？ |

[图示：两个相同的5，稳定排序后它们保持原来的先后顺序]

- ✅ `vector` 的基本使用

# 第1节：冒泡排序（Bubble Sort）

## 核心思想

```
像水里的气泡一样，大的（或小的）元素一点一点"浮"到顶端。
```

### 步骤拆解

```
第1轮：比较第1个和第2个 → 比较第2个和第3个 → …… → 比较第n-1个和第n个
       每比一次，如果前面的比后面的大，就交换。
       一轮下来，最大的元素一定"冒"到了最后面！

第2轮：对前 n-1 个元素重复上面的过程。
       第二大的元素"冒"到了倒数第二的位置。

……

第n-1轮：对前2个元素重复。排序完成！
```

### 伪代码

```
for i = 1 to n-1:                  // 需要进行 n-1 轮
    for j = 0 to n-i-1:            // 每轮比较的范围越来越小
        if a[j] > a[j+1]:          // 前面的比后面的大
            swap(a[j], a[j+1])     // 交换！
```

## 人话类比：「气泡上浮」

想象一杯可乐里的气泡——**小气泡从杯底往上浮，越往上越大**。

每一轮扫描，就像让最大的气泡「浮」到最后面，剩下的继续浮……

## 完整代码

```cpp
// 冒泡排序 —— 从小到大
void bubbleSort(vector<int>& a) {
    int n = a.size();
    // 外层循环：进行 n-1 轮
    for (int i = 0; i < n - 1; i++) {
        bool swapped = false;  // 优化：如果本轮没有交换，说明已经有序
        // 内层循环：每次比较相邻的两个元素
        for (int j = 0; j < n - 1 - i; j++) {
            if (a[j] > a[j + 1]) {      // 前面比后面大 → 交换
                swap(a[j], a[j + 1]);
                swapped = true;
            }
        }
        if (!swapped) break;  // 已经有序，提前结束！
    }
}
```

### 逐行分析

| 行 | 说明 |
|----|------|
| `for (int i = 0; i < n-1; i++)` | 共进行 n-1 轮。为什么是 n-1？因为第 n 轮只剩1个元素，不用比 |
| `for (int j = 0; j < n-1-i; j++)` | 每轮比较的范围缩小：第 i 轮时，最后 i 个已经排好了 |
| `if (a[j] > a[j+1])` | **核心判断**：前面的比后面的大就交换 |
| `swap(a[j], a[j+1])` | C++ 内置的交换函数 |
| `swapped` 标记 | **小优化**：如果一整轮都没有交换，说明已经排好了，直接结束 |

## ⚠️ 常见错误

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| 循环写成 `j < n-1`（没减i） | 每轮都扫全部，浪费时间 | 应写 `j < n-1-i` |
| 忘记 `swapped` 优化 | 已经有序了还在傻傻比较 | 加标记，提前终止 |
| 比较符写成 `a[j] >= a[j+1]` | 稳定性被破坏 | 只写 `>` 保持稳定 |

## 复杂度分析

| 情况 | 时间复杂度 | 说明 |
|------|:---------:|------|
| 最好（已有序） | O(n) | 加了优化后只扫一轮 |
| 平均 | O(n²) | 两层循环嵌套 |
| 最坏（完全逆序） | O(n²) | 每轮都要交换 |

**空间复杂度**：O(1)，原地排序

**稳定性**：✅ 稳定（相等元素不交换）

# 第2节：选择排序（Selection Sort）

## 核心思想

```
每一轮从未排序的部分中「选出」最小（或最大）的元素，放到已排序部分的末尾。
```

### 步骤拆解

```
第1轮：从 a[0]~a[n-1] 中选出最小的，和 a[0] 交换
第2轮：从 a[1]~a[n-1] 中选出最小的，和 a[1] 交换
第3轮：从 a[2]~a[n-1] 中选出最小的，和 a[2] 交换
……
第n-1轮：从 a[n-2]~a[n-1] 中选出最小的，和 a[n-2] 交换
```

## 人话类比：「体育课选人」

体育老师要选人排队：**「你们中间谁最矮？站出来！站第一个。剩下的人里谁最矮？站出来！站第二个……」**

每次从剩下的人里挑最矮的——就是选择排序。

## 完整代码

```cpp
// 选择排序 —— 从小到大
void selectionSort(vector<int>& a) {
    int n = a.size();
    for (int i = 0; i < n - 1; i++) {
        int minPos = i;  // 假设当前位置就是最小的
        // 在 a[i] ~ a[n-1] 中找真正的最小值位置
        for (int j = i + 1; j < n; j++) {
            if (a[j] < a[minPos]) {
                minPos = j;  // 找到更小的，更新位置
            }
        }
        // 把最小值换到前面
        if (minPos != i) {
            swap(a[i], a[minPos]);
        }
    }
}
```

### 逐行分析

| 行 | 说明 |
|----|------|
| `int minPos = i;` | 先假设第 i 个就是最小的 |
| `for (int j = i+1; j < n; j++)` | 扫描剩余元素 |
| `if (a[j] < a[minPos])` | 发现更小的，记录位置 |
| `swap(a[i], a[minPos])` | 把真正最小的换到 i 位置 |

## 选择排序的特点

- **交换次数最少**：每轮最多交换 1 次（共 n-1 次交换）
- **不稳定**：例如 `[5₁, 8, 5₂, 2]` → 第一轮选 2 和 5₁ 交换 → `[2, 8, 5₂, 5₁]`，两个5的相对顺序变了

## 复杂度分析

| 情况 | 时间复杂度 |
|------|:---------:|
| 最好 | O(n²) |
| 平均 | O(n²) |
| 最坏 | O(n²) |

> 无论数据是否有序，选择排序都要老老实实扫完每一轮。

# 第3节：插入排序（Insertion Sort）

## 核心思想

```
像打扑克牌时整理手牌——把新摸到的牌「插入」到已经排好的牌中的正确位置。
```

### 步骤拆解

```
初始：a[0] 自己就是「已排序部分」

第1步：拿 a[1]，在 a[0]~a[0] 中找位置插入 → 已排序部分变成 a[0]~a[1]
第2步：拿 a[2]，在 a[0]~a[1] 中找位置插入 → 已排序部分变成 a[0]~a[2]
……
第n-1步：拿 a[n-1]，在 a[0]~a[n-2] 中找位置插入 → 全部有序！
```

[图示：一排扑克牌，新手牌从右往左找位置插入]

## 完整代码

```cpp
// 插入排序 —— 从小到大
void insertionSort(vector<int>& a) {
    int n = a.size();
    for (int i = 1; i < n; i++) {
        int key = a[i];      // 当前要插入的牌
        int j = i - 1;
        // 从后往前找插入位置，同时把大的元素往后挪
        while (j >= 0 && a[j] > key) {
            a[j + 1] = a[j];  // 往后挪一位，腾出空间
            j--;
        }
        a[j + 1] = key;       // 插入到正确位置
    }
}
```

### 逐行分析

| 行 | 说明 |
|----|------|
| `int key = a[i]` | 把当前元素「抽出来」，像摸到一张新牌 |
| `while (j >= 0 && a[j] > key)` | 只要前面的比 key 大，就往后挪 |
| `a[j+1] = a[j]` | 把大的往后移一格（不是交换，是覆盖！） |
| `a[j+1] = key` | 最后把 key 放到正确位置 |

## 什么时候插入排序最快？

**数据基本有序时**，插入排序几乎是 O(n)！

> 因为 while 循环几乎不进，每轮只比较 1 次就跳过。

这就是为什么 C++ 标准库的 `std::sort` 在数据量较小时会切换到插入排序。

## 复杂度分析

| 情况 | 时间复杂度 |
|------|:---------:|
| 最好（已有序） | O(n) |
| 平均 | O(n²) |
| 最坏（完全逆序） | O(n²) |

**稳定性**：✅ 稳定

# 第4节：计数排序（Counting Sort）

## 核心思想

```
不比较，直接统计！如果数据范围不大，直接数每个值出现了几次。
```

## 人话类比：「投票统计」

班上投票选班长：**不用一个个比谁的票多，直接画「正」字统计每个人的票数，票数从多到少排列就是结果。**

## 步骤拆解

```
1. 统计：用数组 cnt[x] 记录 x 出现了几次
2. 还原：按值从小到大的顺序，根据 cnt[x] 输出 x 共 cnt[x] 次
```

## 完整代码

```cpp
// 计数排序 —— 从小到大（假设数据范围 0 ~ maxVal）
void countingSort(vector<int>& a, int maxVal) {
    int n = a.size();
    vector<int> cnt(maxVal + 1, 0);  // 计数数组，初始全0
    
    // 第1步：统计每个数出现的次数
    for (int i = 0; i < n; i++) {
        cnt[a[i]]++;  // 数 a[i] 出现一次，对应位置 +1
    }
    
    // 第2步：按顺序填回原数组
    int idx = 0;
    for (int val = 0; val <= maxVal; val++) {
        while (cnt[val] > 0) {   // 这个值还有剩余
            a[idx++] = val;       // 填回去
            cnt[val]--;           // 用掉一个
        }
    }
}
```

### ⚠️ 重要限制

计数排序要求：**数据范围不能太大**（比如 maxVal ≤ 10⁷），而且数据必须是**非负整数**。

> 如果要排序 0 到 100 之间的 1000 个数，计数排序是 O(n) 的神器；
> 但要排序 `int` 范围内的 10 个数，计数排序直接爆内存！

## 复杂度分析

| 指标 | 值 |
|------|---|
| 时间 | O(n + maxVal) |
| 空间 | O(maxVal) |

**稳定性**：✅ 稳定（上述写法是稳定的，因为我们按值从小到大依次填回）

# 第5节：STL sort —— 你的主力武器

## 前面学了那么多，为什么比赛只用 sort？

因为在竞赛中，**99% 的情况你不需要自己写排序**——直接用 C++ 标准库的 `sort` 就够了！

```cpp
#include <algorithm>  // sort 在这里
#include <vector>
using namespace std;

vector<int> a = {5, 2, 8, 1, 9};
sort(a.begin(), a.end());  // 从小到大：1 2 5 8 9
```

## sort 的三大优势

| 优势 | 说明 |
|------|------|
| ⚡ 快 | O(n log n)，底层是快速排序+插入排序+堆排序的混合 |
| 🎯 稳 | 标准库实现，不会有边界 bug |
| 🔧 灵活 | 可以自定义排序规则 |

## 自定义排序规则

### 降序排列

```cpp
sort(a.begin(), a.end(), greater<int>());
// 结果：9 8 5 2 1
```

### 自定义比较函数

```cpp
// 按绝对值从小到大排序
bool cmp(int a, int b) {
    return abs(a) < abs(b);
}

sort(a.begin(), a.end(), cmp);
```

### Lambda 表达式（推荐！）

```cpp
// 按字符串长度从小到大，长度相同按字典序
vector<string> words = {"apple", "dog", "cat", "banana"};
sort(words.begin(), words.end(), [](const string& a, const string& b) {
    if (a.size() != b.size()) return a.size() < b.size();
    return a < b;
});
// 结果：dog cat apple banana
```

### 对结构体排序

```cpp
struct Student {
    string name;
    int score;
};

vector<Student> stu = {{"小明", 90}, {"小红", 95}, {"小刚", 90}};

// 先按分数降序，分数相同按名字升序
sort(stu.begin(), stu.end(), [](const Student& a, const Student& b) {
    if (a.score != b.score) return a.score > b.score;
    return a.name < b.name;
});
```

## 比较函数的铁律

> **当 `a` 应该排在 `b` 前面时，返回 `true`。**

❌ 错误写法：`return a <= b;`（会导致崩溃！必须是**严格小于**）

# 第6节：归并排序（Merge Sort）—— 分治的经典

## 核心思想

```
把大问题切成小问题，小问题解决了再合并回来。
```

### 分治三步曲

```
1. 分：把数组从中间切成两半
2. 治：分别对左右两半递归排序
3. 合：把排好的两半合并成一个有序数组
```

[图示：数组不断对半切，切到底再一层层合并回来]

## 人话类比：「两摞扑克牌」

两摞牌都已经从小到大排好了，怎么合成一摞？

> 每次比较两摞牌最上面那张，谁小就把谁拿走。重复直到一摞为空，把剩下的全拿走。

## 完整代码

```cpp
// 合并两个有序数组
void merge(vector<int>& a, int l, int mid, int r) {
    vector<int> tmp(r - l + 1);  // 临时数组
    int i = l, j = mid + 1, k = 0;
    
    // 两路归并：每次取小的
    while (i <= mid && j <= r) {
        if (a[i] <= a[j]) tmp[k++] = a[i++];
        else              tmp[k++] = a[j++];
    }
    
    // 把剩余的直接抄过来
    while (i <= mid) tmp[k++] = a[i++];
    while (j <= r)   tmp[k++] = a[j++];
    
    // 拷回原数组
    for (int t = 0; t < k; t++) {
        a[l + t] = tmp[t];
    }
}

// 归并排序主函数
void mergeSort(vector<int>& a, int l, int r) {
    if (l >= r) return;          // 只剩一个元素，天然有序
    
    int mid = l + (r - l) / 2;   // 防止溢出的写法
    mergeSort(a, l, mid);        // 递归排序左半
    mergeSort(a, mid + 1, r);    // 递归排序右半
    merge(a, l, mid, r);         // 合并左右
}
```

### 逐行分析（合并函数）

| 行 | 说明 |
|----|------|
| `vector<int> tmp(r-l+1)` | 临时数组，存放合并结果 |
| `int i = l, j = mid+1` | i 指向左半开头，j 指向右半开头 |
| `if (a[i] <= a[j])` | **取等号保证稳定性！** |
| `tmp[k++] = a[i++]` | 取了左边的，指针后移 |
| 最后两个 while | 把剩余元素直接抄过去 |

## 复杂度分析

| 指标 | 值 |
|------|---|
| 时间 | O(n log n) —— 所有情况！ |
| 空间 | O(n)（需要临时数组） |

**稳定性**：✅ 稳定（`<=` 保证相等元素保持原序）

# 深度思考

## 1. 排序算法的本质是什么？

> 排序 = **比较 + 交换**。不同算法只是「什么时候比较」「什么时候交换」的策略不同。

| 算法 | 比较策略 | 交换策略 |
|------|---------|---------|
| 冒泡 | 相邻比较 | 发现逆序就交换 |
| 选择 | 全局扫描找最小 | 每轮只交换一次 |
| 插入 | 在有序部分找位置 | 一边找一边挪 |
| 归并 | 分治后合并 | 从临时数组拷回 |

## 2. 什么时候用哪种排序？

| 场景 | 推荐算法 | 原因 |
|------|---------|------|
| 竞赛一般情况 | `sort()` | 又快又稳又省事 |
| 数据量很小（n<20） | 插入排序 | 常数小 |
| 数据范围小（如0~100） | 计数排序 | O(n) 碾压 |
| 需要稳定排序 | 归并排序 / `stable_sort()` | 保持相对顺序 |
| 求逆序对 | 归并排序 | 天然适合统计 |
| 数据基本有序 | 插入排序 | 接近 O(n) |

## 3. 求逆序对 —— 归并排序的神奇应用

**逆序对**：如果 `i < j` 但 `a[i] > a[j]`，则 `(i, j)` 是一个逆序对。

在归并排序的合并过程中：
- 当 `a[j] < a[i]` 时，说明 a[i]~a[mid] 都比 a[j] 大
- 因此逆序对数量增加 `mid - i + 1` 个

```cpp
long long invCount = 0;
// 在 merge 函数中的 else 分支加上：
// invCount += (mid - i + 1);
```

## 4. 易错点总结

| 易错点 | 正确做法 |
|--------|---------|
| 冒泡内循环忘减 i | `j < n-1-i` |
| 选择排序忘记判断 `minPos != i` | 可能导致与自己交换（虽然无害但多余） |
| 插入排序写成交换而非覆盖 | 性能变差！插入排序应该是「挪位」不是「交换」 |
| sort 的比较函数用 `<=` | 必须用 `<`，否则可能崩溃 |
| 归并排序递归不写终止条件 | `if (l >= r) return;` 必须有 |
| 归并排序临时数组大小不够 | 大小应为 `r - l + 1` |
| 计数排序忘记数据范围限制 | 如果 maxVal 太大直接 MLE |

# 例题1：车厢重组（P1116）

> **来源**：洛谷 P1116 | **难度**：⭐
> **题单分类**：模拟 / 排序入门

## 题目大意

一列火车有 N 节车厢，编号为 1~N 但不按顺序排列。每次只能交换相邻两节车厢。问最少需要交换多少次，才能让车厢按 1~N 的顺序排列？

## 思路分析

**这就是冒泡排序的交换次数！**

每次交换相邻两个逆序元素 = 一次冒泡交换。冒泡排序的总交换次数 = 逆序对数量。

### 解法

直接模拟冒泡排序，每次交换计数+1：

## 代码

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;
    vector<int> a(n);
    for (int i = 0; i < n; i++) cin >> a[i];
    
    int ans = 0;
    // 冒泡排序，统计交换次数
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - 1 - i; j++) {
            if (a[j] > a[j + 1]) {
                swap(a[j], a[j + 1]);
                ans++;  // 交换一次，计数+1
            }
        }
    }
    cout << ans << endl;
    return 0;
}
```

### 复杂度分析

- 时间：O(n²)，n ≤ 10000 可能超时，但此题 n ≤ 100，足够
- 空间：O(n)

### 进阶优化

用归并排序求逆序对，O(n log n)，能处理 n ≤ 10⁵ 的数据。

# 例题2：快速排序模板（P1177）

> **来源**：洛谷 P1177 【模板】快速排序 | **难度**：⭐⭐

## 题目大意

给定 N 个整数，将它们从小到大排序后输出。N ≤ 10⁵。

## 思路分析

直接用 STL 的 `sort` 即可。时间复杂度 O(n log n)，完全够用。

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);  // 加速输入输出
    cin.tie(0);
    
    int n;
    cin >> n;
    vector<int> a(n);
    for (int i = 0; i < n; i++) cin >> a[i];
    
    sort(a.begin(), a.end());
    
    for (int i = 0; i < n; i++) {
        cout << a[i] << (i == n - 1 ? '\n' : ' ');
    }
    return 0;
}
```

### ⚠️ 关键细节

1. `ios::sync_with_stdio(false)` + `cin.tie(0)` 是输入输出加速，n=10⁵ 时必须加
2. 如果题目要求**稳定排序**，用 `stable_sort(a.begin(), a.end());`

### 复杂度分析

- 时间：O(n log n)
- 空间：O(log n)（sort 的递归栈）

# 例题3：宇宙总统（P1781）

> **来源**：洛谷 P1781 | **难度**：⭐⭐⭐
> **题单分类**：排序 + 字符串

## 题目大意

有 N 个人竞选总统，每人有若干选票（票数可能非常大，超出 long long 范围！）。票数最多的人当选。如果票数相同，编号小的当选。输出当选者的编号和票数。

## 思路分析

票数太大，不能用 int 或 long long 来比——但可以用**字符串**！

字符串比较的规则：
- 先比长度：长度不同的，长的更大
- 长度相同的，按字典序比较（就是逐位比）

这就是一个**自定义排序**的问题。

## 代码

```cpp
#include <bits/stdc++.h>
using namespace std;

struct Candidate {
    int id;           // 编号
    string votes;     // 票数（用字符串存）
};

int main() {
    int n;
    cin >> n;
    vector<Candidate> cand(n);
    for (int i = 0; i < n; i++) {
        cand[i].id = i + 1;
        cin >> cand[i].votes;
    }
    
    // 自定义排序规则
    sort(cand.begin(), cand.end(), [](const Candidate& a, const Candidate& b) {
        // 票数多的排前面
        if (a.votes.size() != b.votes.size())
            return a.votes.size() > b.votes.size();  // 长度大的在前
        if (a.votes != b.votes)
            return a.votes > b.votes;  // 长度相同，字典序大的在前
        return a.id < b.id;  // 票数相同，编号小的在前
    });
    
    cout << cand[0].id << endl;
    cout << cand[0].votes << endl;
    return 0;
}
```

### 关键点分析

| 行 | 说明 |
|----|------|
| `string votes` | 用 string 存票数，突破整数范围限制 |
| `a.votes.size()` | 先比长度：位数多的票数一定更大 |
| `a.votes > b.votes` | 长度相同，字典序比较就是数值比较 |
| 优先级 | 票数 > 编号，Lambda 表达式清晰表达了优先级 |

### 复杂度分析

- 时间：O(n log n × L)，其中 L 是字符串比较的代价
- 空间：O(n × L)

# 总结

## 一句话核心

> **排序 = 比较 + 交换，竞赛直接用 `sort` + 自定义比较规则。**

## 算法速查表

| 算法 | 时间 | 空间 | 稳定 | 适用场景 |
|------|:---:|:---:|:---:|---------|
| 冒泡 | O(n²) | O(1) | ✅ | 教学演示、求交换次数 |
| 选择 | O(n²) | O(1) | ❌ | 交换次数最少时 |
| 插入 | O(n²) | O(1) | ✅ | 数据基本有序时 |
| 计数 | O(n+k) | O(k) | ✅ | 数据范围不大时 |
| **sort** | **O(n log n)** | O(log n) | ❌ | **竞赛首选！** |
| 归并 | O(n log n) | O(n) | ✅ | 需要稳定排序 / 求逆序对 |

## 适用场景识别

看到以下特征 → 考虑排序：

- 「从小到大输出」「按分数排名」
- 「按某个属性排序后再做处理」
- 排序后可以用双指针/贪心
- 排序消除后效性，简化问题
